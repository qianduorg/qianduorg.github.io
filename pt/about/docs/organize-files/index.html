      Introdução ao diretório

      Por defeito, o Qiandu é mantido com base no Hugo, pelo que o conteúdo é pesquisado por ficheiros Markdown no directório content. Por exemplo:

      • _index.md
        • _index.md
        • getting-started.md
          • _index.md
          • organize-files.md
        • _index.md
        • post-1.md

      • Cada ficheiro _index.md é uma página de índice para a secção correspondente, e outros ficheiros Markdown são páginas normais.

      content
├── _index.md // <- /
├── docs
│   ├── _index.md // <- /docs/
│   ├── getting-started.md // <- /docs/getting-started/
│   └── guide
│       ├── _index.md // <- /docs/guide/
│       └── organize-files.md // <- /docs/guide/organize-files/
└── blog
    ├── _index.md // <- /blog/
    └── post-1.md // <- /blog/post-1/

      Navegação na barra lateral

      A navegação na barra lateral é gerada automaticamente com base na ordem alfabética em que o conteúdo é organizado. Para configurar manualmente a ordem da barra lateral, pode utilizar a definição weight no frontmatter do ficheiro Markdown.

      content/docs/guide/_index.md
      ---
title: Guide
weight: 2
---
      ℹ️
      Recomenda-se que a barra lateral não seja demasiado profunda. Se houver muito conteúdo, considere dividi-lo em várias partes.

      Configurar diretório de conteúdo

      Se precisar de utilizar um diretório diferente para o conteúdo, pode contactar o administrador para definir contentDir no ficheiro de configuração do site para conseguir isso.
