      Sintaxe Markdown

      O Qiandu suporta Markdown para escrever conteúdo, criar listas, etc. Esta página irá mostrar alguns dos exemplos de sintaxe Markdown mais comuns utilizados pelo Qiandu.

      Noções básicas de redução

      Estilo de texto

      StyleSyntaxExampleOutput
      Bold**bold text****bold text**bold text
      Italic*italicized text*`italicized textitalicized text
      Strikethrough~~strikethrough text~~~~strikethrough text~~strikethrough text
      Subscript<sub></sub>This is a <sub>subscript</sub> textThis is a subscript text
      Superscript<sup></sup>This is a <sup>superscript</sup> textThis is a superscript text

      Citar

      Citações em bloco com subscritos:

      Don’t communicate by sharing memory, share memory by communicating.
      — Rob Pike1

      Markdown
      > Don't communicate by sharing memory, share memory by communicating.<br>
> — <cite>Rob Pike[^1]</cite>

[^1]: The above quote is excerpted from Rob Pike's [talk](https://www.youtube.com/watch?v=PAAkCSZUG1c) during Gopherfest, November 18, 2015.

      folha

      As tabelas não são uma especificação básica do Markdown, mas o Hugo suporta tabelas prontas a usar:

      NameAge
      Bob27
      Alice23
      Markdown
      |   Name | Age  |
|--------|------|
|    Bob | 27   |
|  Alice | 23   |

      Inline na tabela Markdown

      ItalicsBoldCode
      italicsboldcode
      Markdown
      | Italics   | Bold     | Code   |
| --------  | -------- | ------ |
| *italics* | **bold** | `code` |

      O Qiandu utiliza o Goldmark para analisar o Markdown. A renderização de Markdown pode ser configurada contactando o administrador da comunidade em markup.goldmark em hugo.yaml. A seguir é apresentada a configuração padrão do Qiandu:

      hugo.yaml
      markup:
  goldmark:
    renderer:
      unsafe: true
  highlight:
    noClasses: false

      Para mais opções, aceda aConfigure Markup。
